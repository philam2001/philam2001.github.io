Golang

I found Golang to be a very interesting language, it being statically typed and it being nit-picky/reliable. It's ideal for system software (run computers hardware and manage computer's resources). It's procedural like python (programs executed in a certain sequence), but it has a light weight concurrent aspect that makes running concurrent processes easily. These are called go routines. It's also more of a functional language compared to python being OO, thus it's better used for applications that need to have multiple operations performed. It's perfect for production in my opinion as its more lightweight and won't take up unnecesary space.
